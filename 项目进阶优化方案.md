## 当前代码问题清单

### 严重：Texture2D 泄漏
```
GameUI.NewTex() 每帧 new Texture2D(1,1)，从不销毁
OnGUI 60fps = 每秒泄漏数百个纹理对象
位置：GameUI.cs、MainMenuUI.cs
```

### 严重：DropItem.Update 每帧 FindObjectOfType
```
DropItem.Update() 每帧调用 FindObjectOfType<PlayerAim>()
20 个掉落物 × 60fps = 1200 次 O(n) 场景搜索/秒
位置：DropItem.cs 第 44 行
```

### 高：new Material 每次创建对象
```
每个敌人/弹珠/火球/掉落物都 new Material(Shader.Find("Standard"))
每局 ~800 次 Shader.Find + ~800 次 new Material
位置：GameManager.CreateEnemyBase、PlayerAim.CreateMarble、DropItem.Spawn、Fireball.Spawn
```

### 高：对象无复用
```
所有对象 CreatePrimitive + Destroy
每局：~150 敌人 + ~300 弹珠 + ~50 火球 + ~250 掉落物
全部触发 GC
```

### 高：FindObjectsOfType 每帧调用
```
GameManager.EnemyCount() 每帧 FindObjectsOfType<Enemy>().Length
位置：GameManager.cs Update()
```

### 中：耦合严重
```
Enemy → GameManager.Instance.AddKill()   直接单例访问
Marble → FindObjectOfType<PlayerHealth>()  场景搜索
ChestItem → FindObjectOfType<PlayerHealth>() 场景搜索（两次）
GameUI ↔ PlayerHealth  双向引用
```

## 阶段七：XLua 热更新（4 新文件 + 3 修改）

### 为什么用 XLua

```
场景：游戏上线后发现 Bug，或想调整游戏平衡

不用热更： 改 C# → 重新编译 → 重新打包 → 提交应用商店审核（1-7 天）
用 XLua：  改 Lua 脚本 → 上传 CDN → 玩家重启游戏自动下载 → 立即生效
```

XLua 相比 HybridCLR 的优势：
- 国内手游项目使用最广泛，面试必问
- Lua 语法简单，策划也能写简单逻辑
- 热更粒度细（单个 .lua 文件即可更新）
- 不依赖 IL2CPP 版本，Unity 2021 完全支持

### 7b. XLuaManager.cs（新建）

```
单例，管理 Lua 虚拟机生命周期

核心职责：
  1. 初始化 LuaEnv（Lua 虚拟机）
  2. 从 CDN 下载最新的 .lua.txt 文件
  3. 将 Lua 脚本加载到虚拟机
  4. 提供 C# 端调用 Lua 的统一入口
  5. 错误处理：Lua 脚本出错时回退到内置默认脚本

关键方法：
  Init()            — 创建 LuaEnv，加载本地兜底脚本
  CheckUpdate()     — 对比 CDN 版本号，有新版本则下载
  LoadScript(name)  — 加载指定 Lua 模块
  CallLua(func, args) — C# 调用 Lua 函数
  Dispose()         — 释放 LuaEnv

Lua 脚本存储路径：
  内置兜底：Application.streamingAssetsPath + "/Lua/"
  热更下载：Application.persistentDataPath + "/Hotfix/Lua/"
```

**enemy/slime.lua.txt** — 敌人 AI（可热更的行为逻辑）
```lua
-- 史莱姆移动 AI
-- 热更场景：上线后发现史莱姆太弱，可以在 Lua 中加入"受伤加速"逻辑
SlimeAI = {}

function SlimeAI:UpdateMovement(self, playerPos, deltaTime)
    -- 计算朝向玩家的方向
    local dx = playerPos.x - self.position.x
    local dz = playerPos.z - self.position.z
    local dist = math.sqrt(dx * dx + dz * dz)
    
    if dist < 0.1 then
        self.velocity = Vector3.zero
        return
    end
    
    local dir = Vector3(dx / dist, 0, dz / dist)
    local speed = self.moveSpeed
    
    -- ★ 热更新增逻辑：史莱姆血量低于一半时加速 50%
    -- 无需重新打包，改这个 Lua 文件上传 CDN 即可
    if self.currentHealth < self.maxHealth * 0.5 then
        speed = speed * 1.5
    end
    
    self.velocity = dir * speed
end

function SlimeAI:OnTakeDamage(self, damage)
    -- 受伤反馈（可在热更中加入闪烁、击退等效果）
    self.currentHealth = self.currentHealth - damage
    if self.currentHealth <= 0 then
        self:Die()
    end
end
```

### 7e. C# 端改造（修改现有脚本）

**SlimeEnemy.cs 改造成 Lua 驱动**

// 改造前（纯 C#）
void FixedUpdate()
{
    Vector3 dir = (player.position - transform.position).normalized;
    rb.velocity = dir * moveSpeed;
}

// 改造后（Lua 驱动）
void FixedUpdate()
{

### 7f. 热更新流程

```
游戏启动
    ↓
XLuaManager.Init()
    ├── 创建 LuaEnv 虚拟机
    ├── 加载内置兜底 Lua 脚本（StreamingAssets/Lua/）
    └── 加载上次下载的热更 Lua 脚本（persistentDataPath/Hotfix/Lua/）
    ↓
XLuaManager.CheckUpdate()
    ├── HTTP 请求 CDN: /version.json
    ├── 对比本地版本号
    ├── 有更新 → 下载新 .lua.txt 文件到 persistentDataPath
    └── 无更新 → 跳过
    ↓
游戏正常运行
    ├── 敌人 AI → CallLua("SlimeAI.UpdateMovement", ...)
    ├── 弹珠特效 → CallLua("MarbleEffects.OnHitEnemy", ...)
    └── 数值读取 → CallLua("Balance.GetEnemyStats", "slime")
    ↓
（下次启动时，新 Lua 脚本生效）
```
